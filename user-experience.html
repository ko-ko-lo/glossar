User Experience (UX)

3.7.2023

Die Aufgabe von User Experience Designern ist es, dafür zu sorgen, dass die besten Entscheidungen für die User, für das Produkt und für das Unternehmen getroffen werden.

Aber was ist eine "User Experience"? (Nicht zu verwechseln mit einem User Interface.)

Stellen wir uns vor, wir haben ein technisches Gerät, das kann ein Smartphone, ein Laptop oder ein Virtual-Reality-Headset sein. Und dann haben wir eine Person, die mit diesem technischen Gerät interagiert.

Immer wenn jemand ein technisches Gerät benutzt, macht die Person eine Experience. Wir bezeichnen die Person als User und sprechen daher von der User Experience.

Übersetzt ins Deutsche bedeutet User Experience Benutzererfahrung.

5 grundlegende UX Richtlinien

Großartige User Experiences entstehen nicht von selbst. Man muss sie bewusst entwerfen. Aber was braucht es, um eine großartige UX zu entwerfen?

Prinzipien sind Wegweiser. Sie stellen sicher, dass die User Experience Designer die besten Entscheidungen für die User und für das Produkt treffen.

Hier sind 5 grundlegende UX Richtlinien:

1. Design für die Zielgruppe

Es besteht kein Zweifel, dass das Alter nur eine Zahl ist.

Aber was ist mit der Tatsache, dass wir in unterschiedlichen Umgebungen aufgewachsen sind - sowohl kulturell als auch technisch. Sind unsere Bedürfnisse nicht unterschiedlich?

Wird zum Beispiel eine Anwendung für Arbeitnehmer erstellt, die in ihrem Berufs- und Privatleben kaum digitale Produkte nutzen? Oder entwickeln wir ein digitales Produkt für Programmierer?

2. Design für den Nutzungskontext

Wie wird das Produkt von der Zielgruppe genutzt? Wo wird das Produkt genutzt?

Entwickeln wir eine Software, die dafür gedacht ist, dass jemand am Schreibtisch sitzt? Oder entwickeln wir eine App, die jemand beim Spazierengehen auf der Straße benutzt?

3. Das Produkt soll einfach zu bedienen und zu erlernen sein

Es gibt ein Mantra im User Experience, das lautet: Du bist nicht der User. Und deshalb muss herausgefunden werden, was der User braucht.

Das bedeutet, dass das Produkt getestet werden muss, denn:
Was man selbst für einfach zu benutzen hält, ist vielleicht nicht das, was die Zielgruppe für einfach zu benutzen hält.

Im User Experience wird sichergestellt, dass ein Produkt so benutzerfreundlich wie möglich ist.

4. Mach das Produkt zufriedenstellend und ansprechend

Was ist für eine spezielle Zielgruppe wichtig, damit es eine ansprechende Experience wird?

Es werden nicht nur Produkte entworfen, die einfach zu bedienen sind, sondern auch solche, die die Menschen gerne benutzen. Sie sollen wiederkommen wollen, um das Produkt immer wieder zu benutzen.

Es muss also herausgefunden werden, was eine Person dazu motiviert, das Produkt beim ersten Mal zu benutzen. Was wird die Person dazu motivieren, es wieder zu benutzen?

5. Berücksichtige die Ziele des Unternehmens

Die Ziele der Organisation müssen berücksichtigt werden, wenn man ein Produkt entwickelt. Es gibt einen Grund für die Existenz des Produkts, wahrscheinlich finanziert jemand das Produkt. Es gibt einen Zweck, der über die Erstellung einer großartigen User Experience hinausgeht.

Es wird zum Beispiel eine Nachrichtenseite entwickelt, und das Unternehmen möchte mit Bannerwerbung Geld verdienen. Man befindet sich also in einem potenziellen Konflikt zwischen dem, was der Benutzer will und braucht, und dem, was das Unternehmen will.

Die Aufgabe eines User Experience Designers ist es, die Interesse des Unternehmens und der Zielgruppe zu beachten, auch wenn das vielleicht etwas schwierig ist.

Die oberste Priorität im User Experience: der Mensch

Egal was passiert, der Benutzer muss immer an der ersten Stelle stehen.
Das mag widersprüchlich klingen: Es sind nicht nur die Ziele des Unternehmens, die in Widerspruch zueinander stehen können.

Es gibt auch andere Dinge:

  • Vielleicht gibt es technologische Einschränkungen
  • oder Zeit und Budget für das Projekt sind begrenzt

Man kann nicht alles tun, was man gerne tun würde, um das User Experience zu gestalten.

Selbst wenn also Entscheidungen getroffen werden, die mit gewissen Kompromissen verbunden sind, muss der User Experience Experte immer die Stimme des Users sein.

Andernfalls wird sich die User Experience verschlechtern.
Wir alle haben wahrscheinlich schon einmal ein Produkt mit einer schlechten Experience genutzt und kennen die damit verbundene Frustration.

User Experience: Aufgaben und Aktivitäten

  • Durchführung von Research und Usability Testing
  • Identifizierung der Benutzerbedürfnisse
  • High-Level-Design mittels Wireframes
  • Arbeit mit Prototypen
  • Erstellung einer Informations-Architektur ↪

Fazit User Experience

Wann immer eine Person ein technisches Gerät benutzt, macht diese eine User Experience. Egal ob eine Rechnung bezahlt wird, ob die Person mit ihren Freunden chattet oder einen Mietauto bucht.

Mit User Experience stellt man sicher, dass diese Menschen eine großartige, reibungslose Benutzererfahrung haben. Mithilfe von Qualitative Research, Quantitative Research und UX Richtlinien können die besten Entscheidungen für die Benutzer und das Produkt getroffen werden.

Das Aufgabengebiet im Bereich User Experience ist vielfältig und hängt von den Produktanforderungen ab: Wird ein neues Produkt entwickelt oder wird an einem bestehenden Produkt gearbeitet?

User Experience (UX) und User Interface (UI) sind miteinander verwandt, aber haben unterschiedliche Bedeutungen.
